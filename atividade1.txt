1. PASTA DE GESTÃO (DOCUMENTAÇÃO DE ENGENHARIA)

1.1. DOCUMENTO DE VISÃO
O projeto Facchini resolve o problema de ineficiência e falta de rastreabilidade na coleta manual de volumes da Facchini Logística através de uma arquitetura leve e flexível baseada em objetos de dados.
- O problema técnico: A dependência de processos físicos gera atrasos e falhas na conferência de carga entre unidades.
- Solução Técnica: Implementação de um Back-end robusto em PHP que utiliza a persistência em Arquivos JSON para armazenamento dinâmico de dados. O sistema utiliza captura via QR Code/Barcode para alimentar o buffer de dados e dispara relatórios automatizados via protocolo SMTP (PHPMailer).
- Inteligência de Dados: O software processa e trata os dados coletados em formato JSON, permitindo uma comunicação ágil entre o leitor e os serviços de notificação para as filiais de destino.

1.2. PRODUCT BACKLOG (REQUISITOS FUNCIONAIS E REGRAS DE NEGÓCIO)
- [RF01] Captura e Serialização: O sistema deve capturar códigos via scanner e serializá-los em um arquivo `items.json` de forma estruturada.
- [RF02] API de Consulta: O Back-end deve prover endpoints que retornam o estado atual da coleta em formato JSON.
- [RF03] Gestão de Destino Duplo: O sistema deve processar a lógica de filial intermediária e final, tratando as informações como metadados do relatório.
- [RF04] Automação de Relatórios: O serviço de Back-end deve compilar os dados do JSON e disparar e-mails para os destinatários envolvidos.
- [RN01] Persistência em Arquivo: Os dados devem ser mantidos no arquivo físico no servidor para garantir a integridade entre sessões.
- [RN02] Reset de Coleta: Após o envio bem-sucedido, o arquivo de itens deve ser limpo ou reinicializado para a próxima operação.

1.3. CRONOGRAMA DE SPRINTS (90 HORAS)
- Sprint 1 (30h) - Estrutura de Dados e Mockups: Definição do Schema JSON e setup da arquitetura de persistência em arquivo.
- Sprint 2 (30h) - Lógica de Manipulação JSON: Desenvolvimento dos métodos de leitura, escrita e exclusão no buffer de arquivos.
- Sprint 3 (30h) - Integração de Mensageria: Implementação do ReportService e fluxo de notificação para filiais via SMTP.

2. PASTA DE DESENVOLVIMENTO (ARQUITETURA TÉCNICA)

2.1. MODELO DE ESTRUTURA DE DADOS (SCHEMA JSON)
Diferente dos bancos de dados relacionais tradicionais, o sistema utiliza uma estrutura de Documento (NoSQL) para maior agilidade:
Exemplo de Estrutura do Objeto `item`:
{
  "id": 1617283940,
  "code": "FAC-UNIT-2024",
  "timestamp": "2024-04-01 10:00:00",
  "scan_count": 1
}
Essa abordagem permite que a inteligência do software trate os dados como objetos nativos, eliminando a sobrecarga de consultas SQL complexas em operações de campo.

2.2. ANÁLISE DE VIABILIDADE TÉCNICA (STACK BACK-END)
- Linguagem: PHP 8.0+. Escolhida pela facilidade de manipulação de fluxos de arquivos (file streams) e funções nativas de codificação/decodificação JSON.
- Persistência: Sistema de Arquivos JSON. Escolhido pela portabilidade extrema e facilidade de depuração em ambientes industriais, onde a simplicidade da estrutura de dados garante rapidez na manutenção.
- Tratamento de Dados: O Back-end atua como um coordenador que mapeia os objetos JSON recebidos do Front-end e os processa para a geração de relatórios de saída.

2.3. MAPEAMENTO DE APIS E SERVIÇOS
- HTML5 QRcode Library: Scanner industrial via interface de câmera.
- Lucide Icons: Biblioteca para sinalização técnica de estados no sistema.
- SMTP Gmail API (via PHPMailer): Serviço para envio criptografado dos dados processados pelo Back-end para as filiais correspondentes.
